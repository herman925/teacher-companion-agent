# Design System: 小小探索家 (Little Explorers — Preschool Theme-Inquiry Course Assistant)

**Scope:** `demo/` web interface. Source of truth for all demo UI work; the design judge and any generation prompts read from here.

**Product name:** 小小探索家：学前教育主题探究课程助手 (EN: *Little Explorers — Preschool Theme-Inquiry Course Assistant*). 小小探索家 is the platform; 陪跑智能体 remains the name of the companion-agent *concept* inside it (glossary-canonical) — the agent still calls itself 我.

## 1. Visual Theme & Atmosphere

**"A patient colleague's desk in a kindergarten — professional hands, child-lit room."** The base register is unchanged: sitting across from an experienced teaching-research mentor at a warm wooden desk with good stationery — calm, literate, unhurried, quietly Lingnan (岭南). Nothing bureaucratic (no form grids, no wizard steppers, no progress bars) and nothing generic-AI (no neon gradients, no glassmorphism, no floating orbs, no emoji as icons).

On top of that base sits a restrained layer of **child-explorer accents** — the 放大镜 logo, footprint motifs, slightly springier micro-motion on brand elements, rounded shapes. The rule for every accent: *a tired teacher must never pay for it*. Accents live at the edges (logo, empty states, dots, chips); reading surfaces (messages, artifact cards, plans) stay serene and professional. If an accent competes with content, the accent loses.

Density is **airy but purposeful** — generous paper-like whitespace around a single conversation column, with structured artifacts (cards) that appear *within* the conversation like handouts passed across the desk. Motion is gentle and physical (see §6): content settles; only brand accents are allowed a small, occasional spring.

Two themes: **浅色 (light — rice-paper day)** and **深色 (dark — evening lesson-prep lamp)**. Dark is a *warm* dark — cocoa and lamplight, never the debug drawer's cool slate. Default follows the system (`prefers-color-scheme`); the choice persists per browser.

All colors live as CSS custom properties on `:root` (light) with a complete `[data-theme="dark"]` override set. Components reference tokens only — never raw hex.

## 2b. Brand & Logo

**Logo: 放大镜里的小脚印** — a child's magnifying glass (round lens, stubby handle tilted 40°) with a single tiny footprint centered in the lens. 探究 made literal: someone small is following tracks. Drawn as inline SVG only (no image assets): lens ring + handle in `--green` (2.5px stroke, round caps), footprint in `--persimmon` (filled, one heel pad + four toe dots — readable at 16px favicon size), an optional faint `--gold` glint arc on the lens at ≥28px sizes.

- **Header lockup (v2):** a taller bar. Far left the hamburger (历史课程, ChatGPT-style; hidden for visitors). Then the logo at 48px beside a **two-line name lockup**: 小小探索家 in ZCOOL KuaiLe 24px `--green` over 学前教育主题探究课程助手 in serif 12px `--ink-faded` letter-spaced — big title and platform subtitle read as one mark with clear hierarchy and a colour step. Course name · stage stays as the faded line beside it. **Center cluster:** 新课程 (visitor), 用户中心 person icon, sun/moon theme toggle. **Far right:** the debug spanner — **role-gated, not channel-forked**: dev channels (local / tunnel / dev instance) always show it; the public instance (`CHANNEL=public` in `.env`, reported via `/api/health`) shows it only to a signed-in role above teacher (admin), so public and dev run identical code and stay merge-synced. Ctrl+` obeys the same rule. UI-gating only — the drawer reveals client-side state, never server secrets. Below 640px the platform subtitle drops and the logo shrinks to 38px.
- **Favicon:** the same SVG as a data URI; footprint must survive 16px.
- **Entrance motion:** on first paint the footprint fades in with a tiny 4px "step" (single, ≤400ms, `power2.out`); the lens does not move. Reduced-motion: plain fade.
- The logo never bounces, spins, or reacts to chat events. It is a mark, not a mascot.

## 2c. 探索天空 (ambient kid layer)

A fixed, pointer-transparent background layer of faint brand motifs — paper boat, pinwheel (head slowly turning), a pair of alternating footprints, the magnifying glass, one gold star — drifting at glacial speeds (9–90s cycles) behind the paper. Opacity whispers: ≤7% light, ≤5% dark. The bar it must clear: *noticed on the third visit, never while reading*. Content always sits above (`.chat-main` z-index); under `prefers-reduced-motion` the sky freezes in place (visible, static); on <720px only the footprints remain. Display type joins the accent layer: the wordmark and the welcome greeting use ZCOOL KuaiLe (rounded, hand-drawn; serif fallback offline). **Agent prose is literary serif** (Noto Serif SC 400, 16.5px/1.9) — a colleague's written note, warmer than UI sans without being playful; question-card texts share it. Teacher bubbles and controls stay sans. Tired-teacher readability is non-negotiable.

The v2 sky carries eleven motifs — boat, pinwheel (turning head), footprint pair, magnifier, star, kite (swaying), cloud (drifting), paper plane (gliding), balloon (rising), building blocks, leaf (turning) — at ~10% opacity light / 7% dark, every one animated on its own 9–120s cycle.

## 3. Typography Rules

- **Display / headings / artifact-card titles:** Noto Serif SC (思源宋体), weight 600–700. A literary serif keys the whole product to 课程故事 culture — teacherly, bookish, dignified.
- **Body / chat / UI controls:** system sans stack — `"PingFang SC", "Microsoft YaHei", "Noto Sans SC", sans-serif`, weight 400; 500 for emphasis and button labels. Body size 16px (15px minimum on mobile), line-height 1.75 for Chinese prose comfort.
- **Code / state fields / debug:** `"JetBrains Mono", Consolas, monospace`, 13px, used only in the debug drawer and for `course_state` field names.
- Letter-spacing: default for Chinese; +0.02em on serif display lines. Never letter-space Chinese body text.
- Chinese punctuation is full-width; line lengths capped at ~38 CJK characters for readability.

## 4. Component Stylings

* **Chat — agent messages:** not bubbles. Full-width blocks on Banyan Wash with a 3px Banyan Deep Green left rule, generously padded (20px), subtly rounded (8px). Reads as a colleague's written note, not a chat app.
* **Chat — teacher messages:** right-aligned soft rectangles (12px radius) on white with a hairline warm border (`#E5DED2`), max-width 75%.
* **Example-answer chips (the anti-form-filling workhorse):** pill-shaped (fully rounded), white fill, 1.5px Lion-Dance Persimmon border, persimmon text; on hover/tap fill warms to `#FBEFE7`; on selection the chip's text is inserted into the input for editing (never submitted directly — the teacher always keeps the pen). Chips wrap in a row beneath the agent's question, staggering in.
* **Artifact cards (切口卡 / 问题池 / 访谈卡 / 目标轴心):** Question-Wall Cream stock, 10px radius, whisper-soft diffused shadow (`0 2px 12px rgba(45,42,38,0.08)`), serif title with a small persimmon seal-style tag for the card type (e.g. 切口卡), body in labeled sections. 待现场确认 items carry a dashed underline and a small faded-ink tag — visibly provisional, not shameful.
* **Closure-loop card:** the round-ending signature. White card, 2px Osmanthus Gold border, four rows each led by a gold number-in-circle: 本轮可以去做 / 建议素材 / 回来告诉我 / 我会继续帮你. Rows reveal sequentially (see §6). No other component may use gold.
* **Harness-refusal notice:** when L3 blocks fabrication or a gate refuses a request (e.g., full plan before evidence), the reply renders normally but carries a slim Muted Brick top rule and a one-line plain-language reason plus the next-evidence task. Courteous, brief, never scolding.
* **Input area:** single rounded field (12px) with warm border, growing to 5 lines; send button is a Banyan Deep Green circle with an ink-brush arrow glyph. A quiet 先跳过 text link sits left of send whenever the agent has asked a question.
* **Settings modal:** settings open as a centered modal (as chat products do), not a side drawer — a paper card (max 720px, 14px radius, soft deep shadow) over a translucent ink scrim. Inside: a slim left nav (通用 / 模型服务 / 教师档案) and one pane at a time; nav items use the rail's row treatment (cream hover, Banyan Wash + green for the active item). Serif title, hairline head rule, close ✕ top-right; Esc and scrim-click dismiss. Below 560px the nav becomes a horizontal row and the card becomes a near-full sheet. The modal stays entirely in the warm family.
* **Settings modal — pane contents.** 通用 owns what a teacher actually decides: the **model choice** (the most important control, so it lives first) plus, when the chosen provider has mainland/international variants, a single 线路 selector — MiniMax and GLM appear once each in the dropdown, and the channel switch picks 国内/国际 underneath (演示模式 stays its own entry). Then **外观** (跟随系统 / 浅色 / 深色 — persisted per browser, defaults to system); the header also carries a **sun/moon toggle** — one click flips light↔dark as an explicit choice, the icon showing the theme you'd switch TO (跟随系统 lives only in this pane). 开发者模式 follows. 模型服务 is purely API plumbing: per-channel keys and model ids, the custom endpoint, and — collapsed at the bottom as 高级 — the 服务器地址 field with an honest explanation: it exists only for static hosting (e.g. GitHub Pages) that must point at a remote proxy; via the tunnel or a local run it stays empty. 教师档案 holds the teacher profile (below). A quiet note records the plan of record: provider choice eventually collapses to 官方服务 (platform-provided, keys server-side) vs 自备密钥 BYOK.
* **Teacher profile (教师档案) fields:** 地区 = 省级 select（31 省市区 + 中国香港 / 中国澳门 / 中国台湾 + 其他）+ 区/县 free text; 年龄段（可选，5 intervals）; 教龄（总）in intervals（0–2 / 3–5 / 6–10 / 11–20 / 20 年以上）; 本园年资 in intervals; 角色 select fitting mainland kindergartens（班主任 / 配班教师 / 保育员 / 年级组长 / 保教主任 / 园内教研员 / 副园长 / 园长 / 实习教师 / 其他）; 任教班级 as checkboxes（小班 / 中班 / 大班 / 混龄 — multiple allowed）; 班额 number; 回应风格 as seven fixed choices（简洁要点 / 温和鼓励 / 详细讲解 / 案例参照 / 提问引导 / 极简速览 / 蓝图共创）. All optional, local-only in the demo; injected read-only into the prompt, never model-writable. When accounts land this pane graduates into the user console (people icon) alongside password and display-name management.
* **Admin data console (`/admin`):** speaks the same design language as the teacher UI — paper page, serif headings, warm hairlines, cream table-row hovers, green primary actions, brick destructive actions. It is an operations surface, so density is higher (a real table, mono only for the JSON record pane), but nothing slate: the cool palette stays quarantined to the debug drawer. Includes a collapsible 使用指南 for teammates (connect via the wizard, password rules, manage/export/delete) at the top.
* **Admin console — clarity rules:** humans are identified by **username（昵称）** everywhere; raw UUIDs appear only inside the full-record JSON pane, as a hover title, and as an off-by-default 用户ID column for when they are genuinely needed. Both tabs carry a live **filter box** — type anything (course title, username, 昵称, demographics, id fragment) and the table narrows as you type, with a visible `x/y` count. Bulk actions and **export follow the filter**: 导出筛选结果 exports exactly the rows you can see, 全部删除 becomes 删除筛选结果 when a filter is active — what you see is what you act on.
* **Admin console — tables are instruments:** every column header sorts (click toggles ↑/↓, indicator shown; default 更新时间 ↓). A 列 ▾ picker toggles columns on and off — including teacher-demographic columns joined from the profile (地区 / 角色 / 教龄 / 任教班级 / 班额) and the 用户ID column — and the chosen set persists in that browser's localStorage (恢复默认 resets). The 数据 tab adds an advanced filter row: an 更新时间 date range plus dropdowns for 地区 / 角色 / 教龄 / 任教班级 whose options are built from the values actually present; all filters AND together with the text box. **When accounts land** the console grows a tab row (数据 / 用户): the 用户 tab lists users (角色 / 状态 / 最近登录), creates accounts registration-free — the one-time temporary password is shown exactly once with a copy button — and offers reset password / disable (brick, two-step) / role change. Every action writes an audit row.
* **用户中心 controls everything (one modal):** the person icon opens THE control surface — left nav 账号/登录 · 教师档案 · 通用 · 模型服务 · 登录设备. There is no separate settings modal or gear anymore. Auth panes (账号 or 登录, 登录设备) appear only when the backend offers accounts and are rebuilt on every open; the settings panes (通用 with 外观/开发者模式, 模型服务 with keys/models/高级) are static and built once. 账号: display name (rules stated gently inline: 全站唯一、半年可改一次、有敏感词过滤) and password change. 教师档案 is always a real editing pane — signed in it syncs to the account, signed out it stays local-only. Same warm family throughout.
* **回应风格 is a promise, not a label:** each of the seven styles maps to an explicit directive injected into the system prompt (single-sourced in `prompt-builder.mjs`), and the UI shows the exact directive as the explanation under the select (and as option tooltips) — the teacher reads precisely what the model is told to do. 极简速览 compresses prose only — structured artifacts, closure loops and safety content stay complete. 蓝图共创 is the blueprint-first trial style (full plan from available info, unhappened child content marked 预设/待现场验证, deliver-then-ask): it is the prompt-level spike for the 预设/生成 rebalance; if the ADR adopts that rebalance the behavior becomes default and this style folds away.
* **Style adherence is checked in three tiers, never by block-and-retry:** (1) deterministic proxies run in L3 as **warn-level** violations (logged, shown in the debug drawer, never blocking, never triggering the L4 rewrite — a retry re-sends the whole prompt and is not worth spending on tone): 极简速览 warns when reply prose runs long; 提问引导 warns when a turn asks nothing. (2) Semantic styles (温和鼓励 / 详细讲解 / 案例参照 / 蓝图共创) are judged, if at all, by a sampled dev-mode LLM judge measuring adherence rate during spikes — findings fix the prompt, not the runtime. (3) block+retry stays reserved for safety rules (fabrication, adult slogans, contract shape). Style is persuasion; safety is law.
* **Question cards (问题卡):** when the agent has several genuine questions, they travel as structured `questions[]` in the turn contract — never as a wall of prose question marks (prose questions are a harness violation once `questions[]` is in play). Each card carries 问 text, a one-line why, 2–3 example-answer chips (same anti-form-filling chip rules — insert, never auto-send), an optional free-text line, and a quiet 跳过 link. **No hard cap on count** — deliberately left open until pilot play shows where the tolerance lies; the harness records a warn at >5 and the session log records per-card answered/skipped so the re-tightening decision is made on data, not taste. Cards render as a horizontal **carousel** (scroll-snap with `scroll-snap-stop: always`, swipe on touch, ‹ › arrows on desktop, dot indicators): exactly one full-width card per view — no sliver of the neighbor, no visible scrollbar (dots/arrows are the only navigation chrome), landing always flush. Card width follows the column at every viewport; on narrow screens the example chips scroll horizontally inside the card. Above the submit bar a 查看全部 toggle flattens the carousel into a stacked list so the teacher can review every answer before committing. The submit bar shows 已答 x / 共 y and one 一起发送 button: answers are packaged into a **single teacher message** (numbered, question text quoted, skipped cards explicitly marked 跳过 — a skip is information too) so the model receives the whole batch in one turn and can light several workflow nodes at once. A lone question keeps the classic single question-block; the carousel appears at two or more.
* **地区 picker (教师档案):** both levels are type-to-search inputs backed by native datalists — 省级 (35 entries incl. 中国香港 / 中国澳门 / 中国台湾 / 其他) and 市／区县 filtered by the chosen province, district-precise (e.g. 广州市番禺区), from a vendored MCA-derived dataset (`demo/src/data/china-regions.json`). Free text stays allowed (中国台湾 has no district list; 其他 is open).
* **Debug drawer:** slides from the right, Study-Lamp Slate, monospace; shows stage, state diff (green/red JSON lines), gate checks with pass/fail ticks, provider + token usage. A keyboard shortcut and a tiny wrench icon open it; teachers should never notice it exists.
* **History rail (历史课程侧栏):** the conversation-list surface (as a chat app has a left rail), shown only when server-side history exists (the persistence tier). It defers to the reading experience — collapsed by default so the single centered column stays undisturbed. A narrow (12px) hot-zone at the left edge reveals it on hover (desktop): it slides in over the left margin on a paper surface with a hairline warm right border and the card shadow (never a green fill — the rail is backdrop, not brand), and slides away when the pointer leaves. A pin toggle keeps it open and reflows the column to its right, and that choice persists across reloads; unpinning returns to hover-reveal. On touch, a header 历史 button opens it as a left sheet. Rows: course title in body sans; the active course is marked with a Banyan Deep Green dot and Banyan Wash fill (never persimmon or gold — those stay reserved). Motion matches the drawers: slide 280ms `power2.out`.
* **History rail — manage & delete:** a 管理 text link flips the rows into a multi-select state (a checkbox per row); a footer then offers 删除所选 (N) and 全部删除. Deletion is destructive, so it is always two-step and inline — never a browser modal dialog: the delete control re-labels to a 确定删除 N 个？ confirm and only completes on a second click (取消, or leaving manage mode, aborts). A single row's ✕ (revealed on hover) uses the same inline confirm. Destructive labels use Muted Brick, consistent with harness-refusal and error color.
* **History rail — button custody:** course actions live in the rail, not the chat header. ＋新课程 and 管理 are rail-footer buttons; when the rail exists (persistence tier) the header carries only the 历史 toggle — no duplicate 新课程. The 历史 button is a true toggle (open ↔ close; when pinned, it unpins and closes). Header 新课程 appears only in the offline/static mode where there is no rail.
* **History rail — course titles:** every course gets a short, course-name-like title automatically — the theme the teacher is actually working on (醒狮, 龙舟…), taken from the course state the model itself fills in; before a theme exists, the first message trimmed hard. Titles are capped at 16 characters — a rail row, not a sentence. Hovering a row reveals ✎ (rename, inline input: Enter saves, Esc cancels) beside ✕; rename is teacher-owned and wins over auto-titling permanently.
* **预设蓝图 card (blueprint, ADR-0003):** the planning lens's mother artifact — a client-numbered collapsible outline, not a mindmap. The model ships a semantic tree only (stable ids + provenance status per node: 已确认 / 老师预设 / AI建议 / 预设·待验证); numbering (1 / 1.1 / 1.2.1), collapse state and status rollups are reconstructed deterministically in `blueprint-util.mjs` — the model never writes display numbers, so revisions re-number for free and later a map view can render the same tree. Modules are `<details>` blocks, open by default; nested branches auto-open only while they contain unconfirmed content — collapse-state doubles as the 亮灯 board (a folded, all-confirmed branch is a done branch; an open one still needs the teacher's eye). Status chips per node + a `n 项待确认` rollup on branch summaries; legend at the card foot. Two-round delivery: v0.1 = 主题判断 + 五步总览 + two network modules with ≤3 gap cards; v0.2 after confirmation = full 预设包 (周计划 / 五类活动方案 / 环境与材料 / 轻量回传), version pill in the card head. Mobile keeps this list forever; the PC 导图 view is a second renderer, never a second source of truth.
* **预设蓝图 导图 view (PC ≥880px):** a 列表/导图 pill toggle in the card head switches to a horizontal tidy-tree SVG built by `blueprint-map-layout.mjs` — pure geometry (subtree-height post-order walk, per-depth columns, parent centered on child span, cubic S-curve edges), zero dependencies, entirely client-side. **Decision record:** mermaid/drawio/excalidraw were rejected — MB-scale payloads, foreign styling that fights the warm tokens, and model-written diagram syntax is exactly the fragility ADR-0003 forbids; ~250 lines of owned layout beats all three. Node status maps to tokens (confirmed = green wash/green stroke; teacher_preset = persimmon wash; ai_suggestion = paper/faded-ink; hypothesis = transparent with dashed persimmon — tentative must LOOK tentative). Long CJK titles truncate at 12 chars with the full `编号 + 标题` in a native tooltip. Clicking a branch folds it (+n badge, canvas shrinks); collapse state is UI state, never data. First render grows in — group fade + local `scaleX` on the rect (never CSS-transform the `<g>`: it would override the SVG positioning attribute), 45ms stagger capped at 900ms, `prefers-reduced-motion` freezes. Overflow pans via native scroll. Mobile never sees this view.

## 5. Layout Principles

- Single conversation column, max-width **720px**, centered; artifact cards may extend to 800px on desktop for breathing room.
- **8pt spacing grid** throughout; message vertical rhythm 24px; card internal padding 24px.
- Mobile-first: at <480px the column is full-bleed with 16px gutters; chips become a horizontal scroll row; the debug drawer becomes a bottom sheet.
- Touch targets ≥44px everywhere (chips included).
- The page header is minimal: product name in serif, course name in faded ink, and the settings (API key/provider) gear — no navigation chrome to compete with the conversation.
- The history rail is the one permitted left surface, and it earns that by staying hidden until asked for (a hover hot-zone, or pinned by explicit choice). First paint is unchanged — one centered column, no competing chrome (§1). Pinned mode reflows the column to the right of a 260px rail rather than overlapping it; hover-reveal overlays the left margin without moving the column.

## 6. Motion (GSAP + CSS transitions)

Character: **settling paper, morning light — with a child's footstep at the edges.** Three registers, strictly tiered:

1. **安定 (content register — the default).** Everything that carries information settles: ease `power2.out`, durations **600–900ms** — unhurried, a colleague laying things on the desk, never an app rushing — stagger 80–160ms. Nothing in this register bounces, pulses, or loops. **Messages slide in from their speaker's side** — teacher from the right (x 32px), agent from the left — two people passing notes across the desk. Artifact card: "dealt onto the desk" (fade + rise 26px, 0.8°→0° settle). Question cue-cards: dealt with a 44px slide from the right, 100ms apart. Closure rows: sequential reveal, 90ms stagger, gold circle draws in (SVG stroke). Visible, unhurried, never violent.
2. **专业 (interaction register — continuous feedback).** Hovers and state changes are CSS transitions, 120–180ms ease-out: cards lift 1–2px with a slightly deeper shadow, chips warm their fill, buttons darken, inputs glow their focus ring. Feedback is immediate and quiet — felt, not watched. Theme switch crossfades surface/text colors over 240ms (one shared `transition` on color/background/border properties; no flash of unthemed content).
3. **童趣 (brand-accent register — rare, small, earned).** Only brand elements may spring, and only once per event: the logo footprint "steps in" on first paint; carousel dots ease their width when becoming active (pill-morph); a chip *press* may scale to 0.97 and release. Spring = `back.out(1.4)` max, scale never past 1.03, never on reading surfaces, never looping. If it can distract from prose, it doesn't ship.

- Question-card carousel: native scroll-snap with `scroll-behavior: smooth`; dots/arrows drive `scrollBy`. Active dot pill-morphs (width 8→20px, 200ms). Card content does not re-animate on swipe — the glide IS the motion.
- Streaming text: no per-token jitter; the caret is a quiet fading bar.
- Status line (正在思考…): three soft ellipsis dots breathing at 1.6s period, opacity 0.3↔0.8 — the one permitted loop, because it signals "working" and disappears with the reply.
- `prefers-reduced-motion`: registers 1 and 3 collapse to simple fades, register 2 keeps instant state changes (no transition), the status loop becomes a static ellipsis, carousel glide becomes instant jumps.

## 7. Voice & Microcopy

UI copy is Simplified Chinese, in the spec's 温和引导 register: warm, concrete, never imperative ("回来时带几句孩子原话和现场照片就很好"). No exclamation marks. No emoji in agent prose. Buttons are verbs (发送 / 先跳过 / 展开卡片). The agent refers to itself as 我, to the teacher as 你, and never says 请注意 or 系统提示.
