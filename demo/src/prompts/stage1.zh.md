# 当前阶段：阶段1 聚焦问题，补齐经验（蓄水期）

本阶段目标：让儿童真实遇见主题，收集真实证据，整理儿童问题池，推导核心驱动问题。**没有儿童原话/作品/体验证据，不能进入目标轴心（阶段2）。**

## WF05 高频情境浸润 + WF05b 真实人物访谈

基于切口卡生成 1 个真实体验活动方案（experience_plan 卡片）：
- 目的（必须说明补哪类儿童经验）、适合年龄、时间地点、人员分工、前期铺垫、观察重点、安全物资、活动后表征建议、回传清单。
- 内嵌访谈卡（interview_card）：为幼儿生成 3 个可问的问题，为教师/家长生成 3 个补充问题。问题必须具体、生活化（问「你划船的时候手会不会疼」，不问「龙舟文化的内涵是什么」）。识别访谈对象（家长、爷爷奶奶、村民、商贩、社区人员）。
- 方案收尾即 round_complete：闭环里明确回传物（儿童原话 3 句、停留点 2 个、照片、卡点 1 个即可——强调最小可接受反馈，降低负担）。

## 教师回传处理（收到现场反馈时）

1. 把教师提供的材料逐条写入 state_delta.children_evidence（kind 分类：child_words / question_wall / artifact / photo / behavior / dwell_point / teacher_observation / interview_record），儿童只用匿名代称（如「男孩A」）。
2. 先回应教师最关心的点，引用具体证据（evidence_refs）。
3. 证据不足时：指出缺什么，给低负担补证据任务；不推进到问题池。

## WF07 儿童问题池整理 + WF07b 文化可能性提示

当 children_evidence 中有儿童原话/表征时，生成 question_pool 卡片：
- 按类型分类：why 为什么类 / how_to 怎么做类 / playful_exploration 好玩探索类 / identity_imitation 身份模仿类 / real_inconvenience 真实不便类 / cultural_understanding 文化理解类。
- 每个问题保留儿童原话，标注来源证据（evidence_refs 必填），剔除没有儿童证据的成人化问题（可列出并说明为何剔除）。
- 对每个有潜力的问题附文化可能性提示（cultural_hint，教师后台专用）：可能藏着的生活经验 / 教师后台可关注的文化线索 / 儿童可继续做的小任务。规则：没有证据时只能说「可能」；每个线索必须转译为儿童小任务；提醒教师这些不是给孩子讲的。

## WF08 核心驱动问题推导

问题池确认后，生成 driving_questions 卡片：2–3 个候选核心驱动问题 + 推荐 + 选择理由 + 年龄适宜性提醒 + 需核实条件。
- 每个候选按五性校验：儿童性、真实性、行动性、公共性、文化可能性。
- 格式参考：「我们怎样帮助/设计/改良/解决……，让……更方便/更理解/更安全/更愿意参与。」
- 选择权交给教师（question 带示例：可以直接选、可以微调、也可以说都不像）。

## 本阶段完成条件（引擎将检查）

children_evidence 非空 且 driving_question 有候选（教师选定更好）→ 可提议进入阶段2。
