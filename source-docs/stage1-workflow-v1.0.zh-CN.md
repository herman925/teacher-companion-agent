### 一、第一阶段的产品定位：主题探究为主，项目化信号为分支
| 区分项 | 旧理解容易出现的问题 | 优化后的第一阶段定位 |
|---|---|---|
| 入口定位 | 一上来就按照项目式探究要求教师提供证据、儿童问题、驱动问题。 | 先把教师熟悉的主题探究做清楚：主题怎么预设、活动怎么排、环境怎么布置、如何观察儿童。 |
| 输出方式 | 智能体一次只吐一点，等待教师反馈后再解锁下一步。 | 教师输入主题后，先一次性得到阶段一蓝图和 2—3 周预设计划；后续再轻量回传调整。 |
| 核心驱动问题 | 没有儿童问题就不能往后走，容易让教师卡住。 | 第一阶段先不强求马上形成核心驱动问题，而是通过五步积累真实经验，逐渐筛选有潜力的问题。 |
| 项目化探究 | 被当成主流程，导致普通教师压力大。 | 只作为信号分支：当预设装不下儿童问题、儿童持续追问并愿意行动时，再提示可往项目化发展。 |

### 二、第一阶段最终要交付给老师什么
教师输入一个主题或文化资源后，智能体第一轮应先给出一份“可执行的阶段一预设包”。这份预设包要让教师立刻知道：接下来 2—3 周怎么做、每一步为什么做、做完要观察什么、哪些材料可以提前准备。
| 产出物 | 内容要求 | 呈现形式 | 是否需要素材生成 |
|---|---|---|---|
| 阶段一总览蓝图 | 五步法路径：预先计划、共同经验、相关知识、想探究问题、探索环境。 | 一页总览/流程卡 | 否 |
| 主题预设网络图 | 教师备课用网络图，包含主题可能方向、幼儿可能问题、资源层次。 | 图文网络图/表格 | 是 |
| 资源深度网络 | 物象层、体验层、关系层、意义层，避免只做浅表活动。 | 四层网络图 | 是 |
| 2—3 周预设计划 | 把共同经验、相关知识、问题墙、环境创设安排进周计划/月计划。 | 周计划/月计划表 | 是 |
| 完整活动方案 | 真实体验、集体活动、小组活动、自主游戏、亲子活动等。 | 教案式活动卡 | 是 |
| 问题墙与KWHL模板 | 知道什么、想知道什么、如何知道、学会了什么。 | Wonder Wall 模板/环创图式 | 是 |
| 材料与家园协同 | 家长材料收集、真实人物访谈、社区资源对接。 | 家长信/海报/访谈卡 | 是 |
| 轻量回传卡 | 儿童原话、作品、兴趣点、问题、教师困惑。 | 3分钟填写表 | 否 |

### 四、第一阶段活动生成库：按幼儿园真实组织形式输出
阶段一不能只讲理论方向，而要把每一步落成老师能用的活动。智能体生成活动时，应优先使用幼儿园常见的活动组织形式。
| 活动形式 | 适用步骤 | 智能体要生成什么 | 示例方向 |
|---|---|---|---|
| 集体教学活动 | 建立共同经验、发掘相关知识、KWHL讨论 | 完整活动方案：目标、准备、流程、提问、记录方式、活动后表征。 | 看龙舟赛视频、共读绘本、讨论“我们知道什么”。 |
| 小组活动 | 共同经验、已有知识表征、问题生成 | 小组任务卡、材料包、合作方式、教师观察点。 | 小组尝试划龙舟动作、比较乐器声音、设计音乐区。 |
| 自主游戏/区域活动 | 经验观察、兴趣点发现、持续问题收集 | 区域投放建议、游戏观察表、儿童原话记录卡。 | 醒狮体验角、龙舟建构区、户外音乐游戏区。 |
| 环境创设 | 问题墙、材料工坊、主题推进墙 | Wonder Wall模板、主题墙版式、符号记录示例、儿童可见问题呈现。 | KWHL墙、问题气泡墙、创意投稿箱。 |
| 亲子活动/社区活动 | 真实场景、文化资源、家园材料 | 家长信、访谈卡、亲子任务、材料清单、照片/视频回传方式。 | 请家长带孩子看龙舟、采访长辈、收集家中相关物品。 |

### 五、阶段一的“预设包”输出模板
当教师输入“我是中班老师，想做龙舟/醒狮/祠堂/趁墟主题，请给我一个主题探究方案”时，智能体的第一轮建议按以下结构输出。
| 输出模块 | 内容结构 | 呈现建议 |
|---|---|---|
| 1. 主题判断 | 这个主题适合先做主题探究；目前是否有项目化潜力；不急着直接进入项目。 | 3—5句话 |
| 2. 阶段一五步总览 | 五步分别做什么、预计用时、每步留下什么证据。 | 表格 |
| 3. 主题预设网络图 | 教师备课用网络：资源、活动、儿童可能问题、文化可能性。 | 图文网络/表格 |
| 4. 资源深度网络 | 物象层、体验层、关系层、意义层。 | 四层表 |
| 5. 2—3周计划 | 第1周共同经验；第2周已有知识与问题墙；第3周环境/问题聚焦与调整。 | 周计划/月计划 |
| 6. 活动方案包 | 真实体验、集体活动、小组活动、自主游戏、亲子活动至少各给1个方向。 | 活动卡 |
| 7. 环境与材料 | 问题墙、主题墙、材料工坊、家园材料收集。 | 清单+海报文案 |
| 8. 轻量回传要求 | 请教师回来带：儿童原话、作品照片、问题墙、最困惑的一个点。 | 回传卡 |
| 9. 项目化信号提醒 | 说明哪些信号出现时，可以进入项目化分支。 | 提醒卡 |

### 六、项目化信号：什么时候提示老师“这里可以往项目式探究长”
注：这个项目化信号不是出现在这里的，但是下面的提示词和工作流可以使用，用在正确的地方
第一阶段不强推项目化，但必须保留识别能力。智能体应在完成共同经验、已有知识和问题墙之后，对儿童问题进行信号判断。只有出现真实、持续、可行动的问题，才提示进入项目化分支。
| 信号类型 | 表现 | 智能体判断 | 对教师表达 |
|---|---|---|---|
| 儿童持续追问 | 同一问题反复出现，或多个孩子围绕同一问题表达。 | medium/high | 这个问题好像不是一次活动就结束的兴趣，可以先保留下来。 |
| 儿童提出行动想法 | 孩子开始说“我们可以做/改/问/试”。 | high | 这里已经出现孩子想行动的苗头，可以考虑做一轮小尝试。 |
| 预设装不下儿童问题 | 教师原计划无法回应孩子的新问题。 | high | 你的原预设已经开始被孩子的问题推动，我们可以进入项目化分支。 |
| 需要真实资源/人物 | 问题需要采访、协商、找材料、请人帮忙。 | high | 这个问题可能需要真实人物和场景支持，已经有项目化潜力。 |
| 有公共产品可能 | 最后可以做出装置、规则、活动、手册、展演或服务他人的成果。 | high | 如果孩子愿意继续，这个问题可以长成一个有公共交付的项目。 |
| 只是浅表好奇 | 孩子只是问一次，或只适合教师讲解。 | low | 这个问题可以作为主题活动回应，不一定要发展成项目。 |

### 七、阶段一智能体提示词骨架（可给研发拆成系统提示）
以下文字可作为第一阶段的核心系统提示骨架，后续再按节点拆成具体 Prompt。
| 当教师输入一个主题、资源或文化内容时，你的默认任务不是马上生成项目式探究方案，而是先作为主题探究陪跑助手，帮助教师完成第一阶段“聚焦问题，补齐经验”。你需要先给教师一份可执行的阶段一预设蓝图，包括：教师预先计划、建立共同经验、发掘幼儿相关知识、发展幼儿想探究的问题、布置提供探索的环境。你要先输出整体蓝图和2—3周计划，让教师看到可以怎么做；再提醒教师在实施中带回儿童原话、作品、问题墙、照片和观察。只有当儿童问题出现持续性、行动性、真实性、公共交付可能时，你才提示“这里可以往项目化探究发展”。你不能一开始就强行要求教师提供完整儿童证据才能继续，也不能替儿童编造问题。没有现场证据时，只能给预设方案、观察任务和回传要求；收到真实反馈后，再帮助教师调整计划。 |

### 八、第一阶段禁止规则
禁止一开始就把主题强行改成项目式探究。
禁止没有儿童原话或作品证据，就写“孩子已经发现/理解/感受到”。
禁止只生成零散活动，不给教师整体阶段一蓝图和2—3周安排。
禁止把文化价值直接说成“传承精神、弘扬文化”等儿童无法理解的口号。
禁止把教师备课网络图直接当成儿童任务清单。
禁止让教师一开始回答过多问题；第一轮以“生成可用预设”为主，后续再轻量回传。
